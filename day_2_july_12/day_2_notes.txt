How Javascript works
=====================
-we have 2 type of Execution Context
     1. Global (anonymous) when Global code is executed
     2. Function :  when a function is executed

-Javascript code gets executed in 2 steps:
    1. Memory Creation  (memory allocation happens for variables and functions)
    2. Code Execution   (code gets executed line by line)



Hoisting
========
-In Memory creation phase, javascript creates memory for all type of variables var/let/const
-But, only for 'var' variables 'undefined' value is assigned, for let/const <value unavailable>
-it allows us to use/access 'var' variables before the actual initialization, this is hoisting.



Temporal Dead Zone
==================
-In Memory creation phase, javascript creates memory for all type of variables var/let/const
-But, only for 'var' variables 'undefined' value is assigned, for let/const <value unavailable>
-let/const variables can be accessed only after they are initialized with actual values
-till the time they are not initialized, they cannot be used(even though they are there in memory)
-the period when we can see the variables in memory but we cannot use them because they are not initialized
 is called TDZ.
